# 5. 마이페이지 & 회원 관리 (My Page & User Management)

## 개요
사용자의 개인정보 조회 및 수정, 비밀번호 변경, 회원 탈퇴 등을 담당하는 도메인입니다.
마이페이지를 통해 사용자는 자신의 계정 정보를 관리할 수 있습니다.

---

- `src/controllers/user/user.ts:1`

**API:**
- `POST /api/users/change-password`
  - Request:
    ```json
    {
      "currentPassword": "oldPassword123!",
      "newPassword": "newPassword456!"
    }
    ```
  - Response:

**특징:**
- **OAuth 로그인 사용자는 비밀번호 변경 불가**

- **비밀번호 변경 후 자동 로그아웃**
  - 보안을 위해 새 비밀번호로 재로그인 필요

  - 비밀번호 변경 (비밀번호가 없음)
  - 이메일 변경 (OAuth 계정 이메일)

## 사용자 정보 필드

### 필수 정보
- `email`: 이메일 (고유, 수정 불가)
- `name`: 이름 (NICE 인증 정보, 수정 불가)
- `phoneNumber`: 전화번호 (수정 가능, NICE 재인증 필요)
- `loginType`: 로그인 타입 (EMAIL, KAKAO, NAVER)

### 선택 정보
- `address`: 주소 (수정 가능)
- `profileImage`: 프로필 이미지 (미구현 가능성)

### 시스템 정보
- `createdAt`: 가입일
- `isVerified`: 본인인증 완료 여부
- `isActive`: 활성화 상태 (탈퇴 시 false)

## 비밀번호 유효성 검사

### 규칙
```typescript
const passwordRegex = /^(?=.*[A-Za-z])(?=.*\d)(?=.*[@$!%*#?&])[A-Za-z\d@$!%*#?&]{8,}$/;

// 검사 항목:
// 1. 최소 8자 이상
// 2. 영문자 1개 이상
// 3. 숫자 1개 이상
// 4. 특수문자 1개 이상 (@$!%*#?&)
```

### 실시간 검증
```typescript
const validatePassword = (password: string) => {
  const errors = [];

  if (password.length < 8) {
    errors.push('비밀번호는 8자 이상이어야 합니다.');
  }

  if (!/[A-Za-z]/.test(password)) {
    errors.push('영문자를 포함해야 합니다.');
  }

  if (!/\d/.test(password)) {
    errors.push('숫자를 포함해야 합니다.');
  }

  if (!/[@$!%*#?&]/.test(password)) {
    errors.push('특수문자를 포함해야 합니다.');
  }

  return errors;
};
```

## 보안 고려사항

### 5.6.1 비밀번호 변경 후 로그아웃
- 비밀번호 변경 완료 시 즉시 로그아웃
- 새 비밀번호로 재로그인 필요
- 보안 강화 (다른 기기에서의 자동 로그아웃)

---

### 5.6.2 회원 탈퇴 시 비밀번호 확인
- 이메일 로그인 사용자는 비밀번호 재입력 필수
- 본인 확인 절차 강화
- OAuth 사용자는 이미 인증된 세션이므로 생략 가능

---

### 5.6.3 개인정보 수정 로그
- 백엔드에서 모든 정보 수정 기록 저장
- 감사 로그 (Audit Log) 유지
- 보안 이슈 발생 시 추적 가능

---

### 5.6.4 전화번호 변경 시 NICE 재인증
- 전화번호 변경은 본인인증 필수
- 타인의 전화번호로 변경 방지
- 본인 소유 휴대폰만 등록 가능


### 정보 수정 시 낙관적 업데이트
```typescript
const updateMutation = useMutation(fetchUpdateUserInfo, {
  onMutate: async (newUserInfo) => {
    // 1. 기존 캐시 백업
    const previousUserInfo = queryClient.getQueryData(['user', 'info']);

    // 2. 캐시 즉시 업데이트 (낙관적 업데이트)
    queryClient.setQueryData(['user', 'info'], newUserInfo);

    return { previousUserInfo };
  },
  onError: (err, newUserInfo, context) => {
    // 3. 실패 시 롤백
    queryClient.setQueryData(['user', 'info'], context.previousUserInfo);
  },
  onSuccess: () => {
    // 4. 성공 시 재조회
    queryClient.invalidateQueries(['user', 'info']);
  },
});
```
